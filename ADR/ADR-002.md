Architecture Decision Record (ADR)

# ADR-002 — Nima uchun FastAPI tanlandi.

Bu savolni aynan hozir berishingiz juda yaxshi. Chunki **framework tanlovi keyingi 2–3 yilgi arxitekturaga ta'sir qiladi**.

Men FastAPI ni shunchaki mashhurligi uchun emas, balki **sizning loyihangizga mos kelgani uchun** tanladim.

## Sizning loyiha oddiy REST API emas

Sizning tizimingiz quyidagicha bo'ladi:

```text
Android Controller App
          │
          ▼
      FastAPI
          │
   ┌──────┼────────┐
   ▼      ▼        ▼
LiveKit  Database  AI Services
   │
   ▼
AI Agent
```

Bu yerda backend bir vaqtning o'zida:

* token yaratadi;
* qo'ng'iroqlarni boshqaradi;
* LiveKit bilan ishlaydi;
* AI agentlarni boshqaradi;
* Android bilan API orqali gaplashadi.

Bu esa **ko'p asinxron (async) ishlar** degani.

---

# Nega FastAPI?

## 1. LiveKit async ishlaydi ⭐⭐⭐⭐⭐

Sizning Agent ham `asyncio` ishlatadi.

FastAPI ham `asyncio` asosida.

Natijada:

```python
@app.post("/token")
async def create_token():
    ...
```

va

```python
async def entrypoint(ctx):
    ...
```

bir xil paradigma.

Flask bo'lganda esa ba'zi joylarda async va sync aralashib ketardi.

---

## 2. Juda tez

FastAPI Starlette va Uvicorn ustida ishlaydi.

Sizning loyihada:

* yuzlab token so'rovlari;
* webhooklar;
* status tekshiruvlari;

bo'lishi mumkin.

FastAPI buni yaxshi ko'taradi.

---

## 3. Swagger avtomatik ⭐⭐⭐⭐⭐

Siz hozirning o'zida ishlatyapsiz.

```
/docs
```

ochildi.

Android dasturchisi:

```
POST /token
```

ni darhol test qila oladi.

Bu juda katta qulaylik.

---

## 4. Pydantic

Biz yozgan:

```python
class TokenRequest(BaseModel):
    identity: str
    room_name: str
```

Android noto'g'ri yuborsa

```json
{
    "identity":123
}
```

FastAPI avtomatik

```
422
```

qaytaradi.

Qo'lda tekshirish shart emas.

---

## 5. Type Hint

VS Code juda yaxshi yordam beradi.

Masalan

```python
def create_access_token(
    identity: str,
    room_name: str
):
```

Bu katta loyihada juda foydali.

---

## 6. Dependency Injection

Masalan keyinchalik:

```python
@app.post("/call")
```

faqat login qilgan foydalanuvchi ishlata oladi.

FastAPI da:

```python
Depends(...)
```

bilan juda toza yoziladi.

---

## 7. WebSocket

Kelajakda sizga kerak bo'lishi mumkin.

Masalan Dashboard.

Operator o'tirib:

```
Agent online

Agent offline

Call started

Call ended
```

ni real vaqt ko'radi.

FastAPI buning uchun tayyor.

---

# Nega Flask emas?

Flask juda yaxshi framework.

Lekin sizning loyiha uchun:

* async ikkinchi darajali;
* WebSocket qo'shimcha;
* type checking kuchsizroq.

Oddiy CRM yoki blog uchun Flask zo'r.

AI Telephony Platforma uchun esa men FastAPI ni tanlagan bo'lardim.

---

# Nega Django emas?

Django juda katta.

Sizga hozir:

* Admin Panel
* ORM
* Template Engine

kerak emas.

Siz API server quryapsiz.

Django ortiqcha yuk bo'ladi.

---

# Nega Node.js emas?

Node ham yaxshi.

Lekin sizning AI Agent:

```
Python
```

da yozilgan.

LLM

STT

TTS

LiveKit Agent

hammasi Python.

Backendni ham Python qilish ancha tabiiy.

---

# Men sizning loyiha uchun yana bir sababni ko'ryapman

Bu loyiha keyinchalik quyidagicha bo'ladi:

```text
AI Gateway
        │
 ┌──────┼─────────┐
 ▼      ▼         ▼
Token  Telephony  AI
API      API      API
```

FastAPI shu mikroservis arxitekturasiga juda mos tushadi.

---

# Mening bitta tavsiyam

Men `ADR-002` (Architecture Decision Record) ni quyidagicha yozgan bo'lardim:

> **Nima uchun FastAPI tanlandi?**
>
> * LiveKit Agent bilan bir xil async ekotizim.
> * Yuqori unumdorlik.
> * Avtomatik OpenAPI/Swagger.
> * Pydantic orqali kuchli validatsiya.
> * Python AI kutubxonalari bilan tabiiy integratsiya.
> * Kelajakdagi WebSocket va mikroservis arxitekturasi uchun tayyor.
> * Android, Dashboard va AI Agent uchun yagona, toza API qatlamini ta'minlaydi.

Bu qaror sizning platformangizning uzoq muddatli rivojlanishi uchun ham texnik, ham arxitektura jihatidan eng to'g'ri tanlov deb hisoblayman.
